Basic Setup and Knowledge for Using the Bot

Requirements to Run the Bot Using Source Code

  • Python version >= 3.8.0
  • Python-pip version >= 21.3.0
  • [Optional] Git; use the latest version
  • A few Python external modules (use the latest version):
    • discord (may use another Discord Python API module due to maintenance issues)
    • packaging
    • requests
    • colorama
    • inputimeout
    • simplejson

Note: You can use python or python3 -m pip install -r requirements.txt to install the required modules. The requirements.txt file is located in the main branch.

Start Setting Up

To set up the bot with minimal configuration, you'll need a Discord user ID and a Discord bot token. This setup will allow the bot to respond to commands from privileged users in Discord chat.

Without additional configuration, you won't be able to use commands like kaboom and webhook spam, but you can use the nuke command. You can configure these settings anytime after completing the basic setup.

If you are a Windows user, you can download the precompiled (.exe) version here.

Build/Run the Bot

If you are using the .exe version for Windows users, you can double-click the .exe file to run it. Skip to the Entering Authentication section to continue setup.

git clone
cd Nuking-Discord-Server-Bot-Nuke-Bot
python3 -m pip install -r requirements.txt
python3 c-realV2.py

Entering Authentication

When you start the bot for the first time, you will be prompted with:

Enter token. >>>

Enter your Discord bot token here. If you don't have a bot token, refer to the Bot Token section.

Note: Support for selfbot tokens is deprecated. While selfbot features are not removed, they may have bugs or be unusable.

After entering the token, you will need to enter your Discord user ID that will be used to command the bot. For instructions on how to obtain your Discord user ID, refer to the How to Get User ID section.

Bot Access Permissions

Discord user IDs are needed for the bot to identify the owner(s) and grant access to commands. Privileged users can manage permissions during runtime using:

.config permissions add <user ID>

Or give all permissions to everyone:

.gap

Check and Add Permissions

There are two ways to check permissions:

  • Use .config permissions list
  • Open the configuration file (usually named default.json)

To modify permissions, you can:

  • Use .config permissions <add|remove> <UserID|line number> (you can find line numbers using .config permissions list)
  • Edit the "permissions" section in the configuration file. Example:
{
    // ...
    "permissions": [
        "349338062485979136", // Discord ID
        "713936815022932079"  // Another Discord ID
    ],
    // ...
}

Setup bomb_message to Use kaboom Command

The Kaboom command is a combination of three commands:

  • .channelbomb - Spam creating text channels
  • .rolebomb - Spam creating roles
  • .categorybomb - Spam creating categories

To use any of the bomb commands, you need to configure the bomb_message feature. There are two ways to configure the bomb_message feature:

  1. Directly edit bomb_message in your configuration file (usually called default.json)
  2. Use .config bomb_messages <wordlist> <operation> <data>

Configuration

Inside the bomb_message feature, you will see two items: random and fixed. These are wordlists used for creating names for roles, categories, and text channels.

Random Wordlist

Used for creating random names. To configure the random wordlist, use:

.config bomb_messages random <word length>

Example: .config bomb_message random 10

Or edit the configuration file directly:

{
    // ...
    "bomb_messages": {
        "random": 10, // note that this has to be a positive integer
        // ...
    },
    // ...
}

Fixed Wordlist

Used for creating names with words randomly chosen from a list provided by the user. To configure the fixed wordlist, use:

.config bomb_messages fixed add <one line of words>

Example: .config bomb_message fixed add C-REAL on top

Or edit the configuration file directly:

{
    "bomb_messages": {
        "fixed": [
            "C-REAL",
            "on",
            "top"
        ],
        // ...
    },
    // ...
}

FAQ

How to Get the Bot Token?

To get a bot token:

  1. Go to the Discord Developer Portal
  2. Create or log into your account
  3. Create a new application
  4. Give the application a name and click "Create"
  5. Select the "Bot" tab in the menu
  6. Click "Add Bot" and confirm
  7. If you receive an error about username availability, change the bot's name in "General Information"
  8. Enable privileged intents
  9. Copy the bot token and start using it

How to Get User ID?

To get your Discord user ID:

  1. Log into Discord
  2. Go to User Settings
  3. Scroll to "Advanced" and enable Developer Mode
  4. Right-click on the user's profile picture and select "Copy ID"
